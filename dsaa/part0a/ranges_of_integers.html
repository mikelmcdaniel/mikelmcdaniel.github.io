Ranges of Integers

In many programming languages, ranges of integers are represented with an inclusive lower bound and an exclusive upper bound. For example, the numbers 0, 1, 2, 3, 4 might be written [0, 5).

The fact that it's a standard or idiomatic way to represent ranges of integers means that if you were to write code using a different idiom, people reading your code are more likely to get confused, so that by itself is a good reason.

Other advantages

There are some minor advantages to writing ranges like this. It doesn't ultimately affect how good your code is. These are just my personal thoughts.

Consecutive ranges have identical lower and upper bounds. For example, the numbers 0, 1, 2, 3, 4 could be the combined ranges [0, 3) and [3, 5).

Calculating the number of integers in the range is cleaner. For example, the numbers 7, 8, 9 are written as [7, 10) and have 10 - 7 = 3 numbers.
