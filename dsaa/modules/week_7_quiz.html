1. Compact Representation

The compact binary tree representation used for (min and max) heaps uses less memory and is faster than node classes with left and right pointers. Why don't we always use the compact binary tree representation?

Answer

The compact binary tree representation still uses space for nodes that are "empty". For example, if have an unbalanced binary search tree where you inserted 1, then 2, then 3, then 4, the tree would have N = 4 nodes and H = 4 height, then the representation would have pow(2, H) - 1 - H = 11 empty slots. The compact binary tree representation would use O(pow(2, H)) memory because only 1 node in each "layer" would be used and all the other nodes would be empty.

2. N-ary Compact Representation Child

The compact binary representation allows you to calculate the index of a "left" child (the 0th child) and "right" child (the 1st child). What if we used the same compact representation for an N-ary tree? Note: In a binary tree, N = 2.

How would we calculate the jth child (where 0 <= j && j < N) in an N-ary tree, given an index i?

Answer

The jth child index of node i in an N-ary tree is N * i + j + 1.

3. N-ary Compact Representation Parent

The compact binary representation allows you to calculate the index of the parent. What if we used the same compact representation for an N-ary tree? Note: In a binary tree, N = 2.

How would we calculate the index of the parent, given an index i?

Answer

The parent index of node i in an N-ary tree is (i - 1) / N, using integer division.

4. Min vs Max Heap

What's the difference between a min heap and a max heap?

Answer

In a min heap, every node is smaller than (or equal to) all it's children nodes, so the root is the smallest node. In a max heap, the opposite is true: every node is larger than (or equal to) all it's children.

Answer

fifth_smallest_element.cpp

#include <cstddef>
#include <functional>
#include <iostream>
#include <queue>
#include <vector>

using std::cout;
using std::endl;
using std::greater;
using std::priority_queue;
using std::size_t;
using std::vector;


template <typename T>
class VectorElementLess {
  const vector<T>& elements;
public:
  VectorElementLess(const vector<T>& elements) : elements(elements) {}
  bool operator()(size_t left_index, size_t right_index) {
    return elements[left_index] < elements[right_index];
  }
};

// Given a vector of elements in any order, return the 5th smallest element
// For example:
//   std::vector<char> chars = {'a', 'b', 'c', 'd', 'e', 'f', 'g', 'h', 'i'};
//   assert(fifth_smallest_element(chars) == 'e');
template <typename T>
const T& fifth_smallest_element(const vector<T>& elements) {
  VectorElementLess<T> cmp(elements);
  priority_queue<
      size_t, vector<size_t>, VectorElementLess<T>
    > q(cmp);
  for (size_t i = 0; i < 5; ++i) {
    q.push(i);
  }
  for (size_t i = 5, i_end = elements.size(); i < i_end; ++i) {
    q.push(i);
    q.pop();
  }
  return elements[q.top()];
}

// This function is very similar to the one above, EXCEPT that it returns a
// copy of the value (T) instead of a const T& to the value inside elements.
template <typename T>
T fifth_smallest_element_value(const vector<T>& elements) {
  priority_queue<T> q;
  auto it = elements.begin();
  for (int i = 0; i < 5; ++i) {
    q.push(*it);
  }
  for (auto it_end = elements.end(); it != it_end; ++it) {
    q.push(*it);
    q.pop();
  }
  return q.top();
}

int main(int argc, const char* argv[]) {
  vector<char> chars = {
      'x', 'y', 'z', 'a', 'b', 'c', 'd', 'e', 'f', 'g', 'h'};
  cout << fifth_smallest_element(chars) << endl;
  cout << fifth_smallest_element_value(chars) << endl;
  return 0;
}

6. Heap Sort Space Big-O

Heap sort takes an array (or vector or other similar container) and sorts it in place. What's the big-O of the amount of memory it uses (not including the memory of the input/array/vector/etcetera itself)?

Answer

O(1).
