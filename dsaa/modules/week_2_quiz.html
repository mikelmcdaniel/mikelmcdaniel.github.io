Amortized Big-O of std::vector::pop_back

The amortized big-O of calling std::vector::pop_back in a vector with n elements is O(n) / n = O(1) because std::vector::pop_back never resizes the array. What would the amortized big-O of calling std::vector::pop_back be if it shrunk by half every time some_vector.size() * 4 < some_vector.capacity()?

Big-O of std::vector::pop_back

The big-O of calling std::vector::pop_back one time in a vector with n elements is O(1) because std::vector::pop_back never resizes the array. What would the big-O of calling std::vector::pop_back one time be if it shrunk by half every time some_vector.size() * 4 < some_vector.capacity()?

Reminders

#include <chrono>

// ...

typedef std::chrono::time_point<std::chrono::system_clock> time_point;

struct Reminder {
  // What we're supposed to remind the user about
  string reminder;
  // When we should remind them
  time_point time;
};

// ...

// Insert the reminder so that reminders are sorted by reminders->time.
void add_reminder(
    Collection<Reminder>* reminders, const Reminder& new_reminder) {
  for (auto it = reminders->begin(); it != reminders->end(); ++it) {
    if (it->time >= new_reminder->time) {
      reminders->insert(it, new_reminder);
      return;
    }
  }
}

// Delete reminders that reminders that are older than the current time.
void delete_old_reminders(Collection<Reminder>* reminders) {
  time_point now = std::chrono::system_clock::now();
  for (auto it = reminders->begin(); it != reminders->end(); ++it) {
    if (it->time < now) {
      it = reminders->erase(it);
    }
  }
}
